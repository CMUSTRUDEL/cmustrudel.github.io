AI's Impact on the Software Development Lifecycle

AI IDEs or Autonomous Agents? Measuring the Impact of Coding Agents on Software Development

Shyam Agarwal1, Hao He1, Bogdan Vasilescu1
Paper Code & Data

Speed at the Cost of Quality: How Cursor AI Increases Short-Term Velocity and Long-Term Complexity in Open-Source Projects

Hao He1, Courtney Miller1, Shyam Agarwal1, Christian Kästner1, Bogdan Vasilescu1
Paper Code & Data
1Carnegie Mellon University

Overview of Research

We've all heard developer stories and marketing claims about AI coding tools speeding up software development. And they do: We've studied this empirically using causal inference techniques across a large sample of GitHub repositories, and the evidence is strong — development velocity (commits, lines added) increases when adopting modern AI coding assistants.

But here's what was less well understood: this velocity increase tends to be short-lived, and tends to come at the cost of code quality and technical debt. Static analysis warnings and code complexity go up substantially and never recover during our observation window.

What We Studied

We analyzed over 800 open-source GitHub repositories that adopted AI coding tools, comparing them to carefully matched control repositories. We looked at two types of AI adoption: projects that jumped straight to autonomous coding agents (like Claude, Devin, or Cursor's Agent mode), and projects that first used traditional AI assistants (like GitHub Copilot) before adopting agents.

Using advanced causal inference methods (difference-in-differences with staggered adoption), we tracked what happened to these projects over time, measuring both development velocity (how fast code gets written) and software quality (static analysis warnings, code complexity, duplication, and documentation).

What We Found

Impact of AI coding tools on software development metrics over time. The top row shows Cursor AI adoption, while the bottom row shows broader autonomous agent adoption. Blue lines represent projects without prior AI tool usage, orange lines show projects that previously used AI IDEs.

The results reveal a striking trade-off. For projects adopting coding agents as their first AI tool, we see impressive velocity gains—commits increase by around 45% in the months right after adoption. But for projects that already used AI IDEs, the productivity boost is minimal or nonexistent, suggesting diminishing returns from adding more AI assistance.

More concerning: code quality degrades across the board, regardless of prior AI experience. Static analysis warnings increase by roughly 18%, and cognitive complexity rises by 39%. These aren't temporary blips—the quality issues persist for the entire six-month observation period, indicating sustained technical debt accumulation.

Why This Matters

These findings challenge the "more AI equals better" narrative. Yes, AI coding tools can accelerate development, especially for teams new to AI assistance. But the benefits fade quickly if you're already using AI tools, and the quality costs are universal and persistent.

For software teams, this means: adopt AI tools strategically, not indiscriminately. The first AI tool you use might give you a real productivity boost. But stacking AI agent upon AI assistant might just give you faster technical debt. Either way, you'll need stronger code review processes and quality safeguards to catch the complexity and issues that AI-generated code tends to introduce.

Both papers were published at the International Conference on Mining Software Repositories (MSR) 2026, part of ongoing research at Carnegie Mellon University's STRUDEL Lab to understand how AI tools are actually changing software development—not just in controlled experiments, but in real-world open-source projects.
