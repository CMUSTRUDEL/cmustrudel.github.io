AI's Impact on the Software Development Lifecycle

AI IDEs or Autonomous Agents? Measuring the Impact of Coding Agents on Software Development

Shyam Agarwal1, Hao He1, Bogdan Vasilescu1
Paper Code & Data

Speed at the Cost of Quality: How Cursor AI Increases Short-Term Velocity and Long-Term Complexity in Open-Source Projects

Hao He1, Courtney Miller1, Shyam Agarwal1, Christian Kästner1, Bogdan Vasilescu1
Paper Code & Data
1Carnegie Mellon University

Overview of Research

We've all heard developer stories and marketing claims about AI coding tools speeding up software development. And they do: We've studied this empirically using causal inference techniques across a large sample of GitHub repositories, and the evidence is strong — development velocity (commits, lines added) increases when adopting all the modern AI coding assistants.

What was less well understood empirically is that this velocity increase tends to be short-lived, and tends to come at the cost of code quality and technical debt: Static analysis warnings and code complexity go up substantially and never recover during our observation window.

Impact of AI coding tools on software development: velocity increases but code quality decreases

These findings have important implications for software engineering practice. While AI coding assistants can boost short-term productivity, teams must be aware of the potential long-term costs to code maintainability and quality. Our research provides empirical evidence to inform better strategies for integrating AI tools into software development workflows.

Both papers were published at the International Conference on Mining Software Repositories (MSR) 2026, part of ongoing research at Carnegie Mellon University's STRUDEL Lab to understand the socio-technical implications of AI in software engineering.
